Strix supports multiple LLM providers through a dual architecture:

- **Native SDK providers** for OpenAI and Anthropic — direct SDK integration with full streaming, prompt caching, and extended thinking support
- **LiteLLM compatibility layer** for 100+ other providers via [LiteLLM](https://docs.litellm.ai/docs/providers)

## Configuration

Set your model and API key:

| Model             | Provider      | Configuration                    | SDK        |
| ----------------- | ------------- | -------------------------------- | ---------- |
| GPT-5.4           | OpenAI        | `openai/gpt-5.4`                 | Native     |
| Claude Sonnet 4.6 | Anthropic     | `anthropic/claude-sonnet-4-6`    | Native     |
| Gemini 3 Pro      | Google Vertex | `vertex_ai/gemini-3-pro-preview` | LiteLLM    |

```bash
export STRIX_LLM="openai/gpt-5.4"
export LLM_API_KEY="your-api-key"
```

## Native SDK Providers

Models prefixed with `openai/` or `anthropic/` are routed to native SDK providers:

- **OpenAI** — Uses `AsyncOpenAI` with streaming, usage tracking, and cost calculation
- **Anthropic** — Uses `AsyncAnthropic` with prompt caching, extended thinking, and streaming tool calls

All other prefixes (e.g. `vertex_ai/`, `bedrock/`, `ollama/`) fall through to LiteLLM.

## Local Models

Run models locally with [Ollama](https://ollama.com), [LM Studio](https://lmstudio.ai), or any OpenAI-compatible server:

```bash
export STRIX_LLM="ollama/llama4"
export LLM_API_BASE="http://localhost:11434"
```

See the [Local Models guide](/llm-providers/local) for setup instructions and recommended models.

## Model Format

Use the `provider/model-name` format:

```
openai/gpt-5.4
anthropic/claude-sonnet-4-6
vertex_ai/gemini-3-pro-preview
bedrock/anthropic.claude-4-5-sonnet-20251022-v1:0
ollama/llama4
```
